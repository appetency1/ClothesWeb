## 三、技术验收

### 3.1 架构规范

| 验收项 | 通过标准 |
|--------|---------|
| MVC分层 | 代码严格按controller/service/dao/model分包，无跨层调用 |
| 前后端分离 | 前端Vue独立运行，后端Servlet独立部署，通过HTTP/JSON通信 |
| 接口RESTful风格 | 接口路径语义清晰，GET查询/POST提交，返回统一JSON格式 |

### 3.2 数据库规范

| 验收项 | 通过标准 |
|--------|---------|
| 表结构完整 | 6张表全部创建，字段类型、长度、约束与文档一致 |
| 外键关联 | t_clothes.type_id → t_type.id；t_cart.cloth_id → t_clothes.id 等 |
| 初始数据 | t_type、t_size有初始数据；至少有一个管理员账号 |
| 字符集 | 使用utf8mb4，支持中文和emoji |

### 3.3 安全规范

| 验收项 | 通过标准 | 验证方法 |
|--------|---------|---------|
| Token认证 | 登录接口返回token，受保护接口必须携带token | 不带token访问购物车，返回401 |
| 权限控制 | 普通用户无法访问后台接口 | 用普通用户token调后台接口 |
| SQL注入防护 | 全部使用PreparedStatement，无字符串拼接SQL | 代码审查 |
| 密码存储 | 密码建议MD5加密存储（教学版可明文，但建议加密）| 查看数据库password字段 |

### 3.4 接口规范

| 验收项 | 通过标准 |
|--------|---------|
| 响应格式统一 | 所有接口返回 `{code, message, data}` 结构 |
| 跨域处理 | 前端能正常调用后端接口（开发环境配置CORS或代理）|
| 错误处理 | 异常情况下返回有意义的错误信息，不暴露堆栈 |
